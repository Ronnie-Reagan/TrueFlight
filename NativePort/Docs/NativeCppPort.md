# TrueFlight Native Runtime

This repo now includes a standalone C++ runtime for `TrueFlight`.

## Current native scope

- SDL3 window/input and Vulkan-backed rendering through the SDL GPU API.
- Native HUD composited as a texture over the Vulkan scene.
- Ported quaternion/vector math.
- Ported STL and GLB/GLTF geometry loading plus model normalization.
- Ported Vulkan terrain/model/cloud rendering with depth testing, alpha-blended clouds, and fog.
- Ported flight dynamics core, propulsion, atmosphere, and rigid-body integration.
- Ported procedural terrain SDF field with water, biome coloring, terraces, caves, tunnels, skirts, dynamic crater support, and minimap sampling.
- Ported native world chunk persistence/wire groundwork for edited terrain deltas, volumetric overrides, crater saves, and explicit tunnel-seed replay.
- Ported wind-driven cloud group simulation.
- Ported procedural engine, wind, and water-adjacent ambience audio through SDL3 audio streams.
- Chase/cockpit camera toggle.
- Native HUD with debug flight data, minimap, geo readout, crosshair, control indicator, and pause overlay.
- Native pause tabs for main actions, camera/settings, flight tuning, terrain generation/runtime tuning, asset browsing, HUD visibility, controls, and help.
- Automatic native preference save/load for exposed UI settings, flight-model tuning values, terrain tuning values, and the selected aircraft model.

The native executable currently uses `portSource/Assets/Models/DualEngine.glb` when available, falls back to `portSource/Assets/Models/DualEngine.stl`, and then falls back to a cube if those loads fail. The pause `Assets` tab can reload `portSource/Assets/Models`, swap to discovered STL/GLB/GLTF assets, and accept drag-and-drop model files at runtime. Native settings are stored in SDL's pref path as `native_settings.ini`.

## Not yet ported

- Networking transport, replication, and a native dedicated server runtime.
- Voice.
- Full glTF material/texture parity with the Lua importer.
- Terrain streaming/SDF worker parity with the Lua chunk system.
- Native parity for the full Lua settings/character/paint menus.
- Lua-side test harness parity.

## Build

The build now compiles SPIR-V shaders with `glslangValidator`. On Windows this is expected to come from the Vulkan SDK, and the CMake setup will auto-detect common `C:\VulkanSDK\...` installs.

If `cmake` is on `PATH`:

```powershell
cmake -S NativePort -B build/native -G "Visual Studio 18 2026" -A x64
cmake --build build/native --config Release
```

You can also use the scripted wrapper from the workspace root:

```powershell
python NativePort\build_trueflight.py
```

If you switch between different `cmake.exe` installs, use a fresh build directory such as `build/native` or delete the old cache first. Reusing a cache generated by another CMake distribution can break regeneration on Windows.

If `cmake` is not on `PATH`, use the Visual Studio bundled executable:

```powershell
& "C:\Program Files\Microsoft Visual Studio\18\Insiders\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe" -S NativePort -B build/native -G "Visual Studio 18 2026" -A x64
& "C:\Program Files\Microsoft Visual Studio\18\Insiders\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe" --build build/native --config Release
```

## Run

```powershell
.\build\native\Release\TrueFlight.exe
```

## Controls

- `W` / `S`: pitch down / up
- `A` / `D`: roll left / right
- `Q` / `E`: yaw
- `Up` / `Down`: throttle up / down
- Mouse: flight yoke input
- Mouse wheel: throttle
- `Ctrl` + Mouse wheel: pitch trim (`Up` = nose down)
- Right mouse: zoom
- `Space`: air brakes
- `C`: toggle chase/cockpit camera
- `M`: tap to toggle minimap, hold + `Up` / `Down` to change map zoom
- `F3`: toggle debug overlay
- `R`: reset aircraft
- `Esc`: pause / resume

## Pause UI

- `Main`: resume, reset flight, camera, map, debug, quit.
- `Settings`: adjust FOV, mouse sensitivity, invert-Y, crosshair visibility, map orientation, and native audio enable/volume controls.
- `Tuning`: adjust mass, thrust, lift, drag, pitch stability, pitch/roll softening, control loading, control-surface rates, trim assist, and ground-friction values live.
  Auto trim is exposed here and now defaults to off in the native runtime so neutral stick does not keep forcing a pull-up trim state. The native flight model now also uses input shaping and rate-limited control surfaces so the aircraft does not react like a direct stick-to-force arcade mapping.
- `Terrain`: adjust the native terrain seed, chunk/radius coverage, terrain quality, LOD radii, heightfield shaping, cave/tunnel carving, and mesh/skirt limits live. The list scrolls around the current selection.
- `Assets`: swap to the builtin cube, load scanned STL/GLB/GLTF assets, or refresh the asset catalog.
- `HUD`: toggle debug, throttle, control, map, and geo modules.
- `Controls` / `Help`: quick reference pages for the current native bindings and workflow.

While paused:

- `Tab` or `H`: switch tabs
- `W` / `S` or `Up` / `Down`: move selection
- `A` / `D` or `Left` / `Right`: adjust the selected setting or tuning value
- `Enter` / `Space`: activate the selected row or reset the selected row to default
- `Backspace`: restore the current settings/tuning/terrain/HUD page to native defaults
- `F5`: rescan `portSource/Assets/Models` from the `Assets` tab
- Mouse:
  Left-click tabs to switch.
  Left-click `Main` and `Assets` rows to activate them.
  In `Settings`, `Tuning`, `Terrain`, and `HUD`, left-click the left half of a row to decrease/toggle one way, left-click the right half to increase/toggle the other way, right-click to reset the row, and use the mouse wheel to adjust the selected row.
